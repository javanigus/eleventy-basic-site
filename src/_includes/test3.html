Test override output folder

Normally, the output (build) folder is /_site. That can be overridden for specific pages. This page is output in the _includes folder by adding

permalink: "_includes/test3.html" permalinkBypassOutputDir: true

to the frontmatter.

Note that if you run this using the --serve flag, then you'll end up in an infinite loop, constantly rebuilding all pages because 11ty will keep detecting a change to the file system when it writes files to locations that are not ignored, e.g. the _site folder. To fix this, add a directive in .eleventy.js to ignore the page built by this template, e.g.

eleventyConfig.ignores.add("_includes/test3.html");
